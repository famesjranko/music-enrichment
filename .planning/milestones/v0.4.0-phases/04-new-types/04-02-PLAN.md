**MusicBrainzApi.kt** - Add 2 new methods:

1. `browseReleaseGroups(artistMbid: String, limit: Int = 100)`: Calls `GET /release-group?artist={mbid}&type=album|ep|single&fmt=json&limit={limit}`. Returns `List<MusicBrainzReleaseGroup>` via `MusicBrainzParser.parseReleaseGroups(json)`.

2. `lookupArtistWithRels(mbid: String)`: Calls `GET /artist/{mbid}?fmt=json&inc=tags+url-rels+artist-rels`. Returns `MusicBrainzArtist?` via `MusicBrainzParser.parseLookupArtist(json)`. NOTE: The existing `lookupArtist` only uses `inc=tags+url-rels`. This new method adds `artist-rels` to also get band member relationships. This is separate so existing artist lookups don't incur the extra API response overhead.

**MusicBrainzParser.kt** - Add 4 new parse methods:

1. `parseBandMembers(json: JSONObject): List<MusicBrainzBandMember>` - Parse `relations[]` array from artist lookup response. Filter for entries where `type == "member of band"` (incoming direction, artist is the band). Each relation has `artist.name`, `artist.id`, `attributes[]` (role info like "lead vocals"), and `begin`/`end` dates. For "member of band" relations, the member info is in `relations[].artist` and the role/instrument is in `relations[].attributes[]`. Set `role` to the first attribute if present, or null.

2. `parseReleaseGroups(json: JSONObject): List<MusicBrainzReleaseGroup>` - Parse `release-groups[]` array from browse response. Extract `id`, `title`, `primary-type`, `first-release-date`.

3. `parseMedia(json: JSONObject): List<MusicBrainzTrack>` - Parse `media[]` array from release lookup response. Each medium has a `tracks[]` array. Flatten all tracks across all media. Extract `title`, `position` (from track's `position` field), `length` (milliseconds), and recording `id` (from `recording.id` if present, else `id`).

4. `parseUrlRelations(json: JSONObject): List<MusicBrainzUrlRelation>` - Parse `relations[]` array from artist lookup response. Filter for entries where `target-type == "url"`. Extract `type` (e.g., "official homepage", "bandcamp", "youtube", "social network", "wikidata", "wikipedia") and `url.resource`. Exclude "wikidata" and "wikipedia" types (already handled by identity resolution).

**MusicBrainzMapper.kt** - Add 4 new mapper methods:

1. `toBandMembers(members: List<MusicBrainzBandMember>): EnrichmentData.BandMembers` - Map each to `BandMember(name, role, activePeriod = beginDate-endDate or beginDate-present if not ended, identifiers with musicBrainzId)`.

2. `toDiscography(groups: List<MusicBrainzReleaseGroup>): EnrichmentData.Discography` - Map each to `DiscographyAlbum(title, year = firstReleaseDate?.take(4), type = primaryType, identifiers with musicBrainzReleaseGroupId)`.

3. `toTracklist(tracks: List<MusicBrainzTrack>): EnrichmentData.Tracklist` - Map each to `TrackInfo(title, position, durationMs = lengthMs, identifiers with musicBrainzId)`.

4. `toArtistLinks(relations: List<MusicBrainzUrlRelation>): EnrichmentData.ArtistLinks` - Map each to `ExternalLink(type = relation.type, url = relation.url)`.

2. In `enrichArtist()`, add routing for new types. The method currently builds an artist result for any type. Add explicit handling BEFORE the existing artist lookup logic:

For BAND_MEMBERS: Need `lookupArtistWithRels(mbid)` (which includes `artist-rels`). If no mbid, search first, get mbid, then lookup. Parse band members from the relations. If members list is empty, return NotFound. Otherwise return Success with `MusicBrainzMapper.toBandMembers(members)`.

For ARTIST_DISCOGRAPHY: Need `browseReleaseGroups(mbid)`. If no mbid, search first. Return Success with `MusicBrainzMapper.toDiscography(groups)`.

For ARTIST_LINKS: Need the URL relations from `lookupArtist(mbid)` (existing method already has url-rels). Parse with `MusicBrainzParser.parseUrlRelations(json)`. But the current API returns parsed `MusicBrainzArtist` not raw JSON. Two approaches: (a) have `lookupArtistWithRels` return the raw JSON too, or (b) add URL relations to `MusicBrainzArtist` model. Go with approach (b): add `urlRelations: List<MusicBrainzUrlRelation> = emptyList()` to `MusicBrainzArtist` data class. Update `parseLookupArtist` (and `parseArtistObject`) to also call `parseUrlRelations` and populate the field. Then in the provider, after lookup, call `MusicBrainzMapper.toArtistLinks(artist.urlRelations)`.

3. In `enrichAlbum()`, add routing for ALBUM_TRACKS: Need a full release lookup (the media array is in the release lookup). Parse with `MusicBrainzParser.parseMedia(json)`. Similar to ARTIST_LINKS above, need to add tracks to the model. Add `tracks: List<MusicBrainzTrack> = emptyList()` to `MusicBrainzRelease`. Update `parseLookupRelease` to also call `parseMedia(json)` and populate the field. Add `&inc=media+recordings` to the release lookup URL (in addition to existing inc params). Then in enrichAlbum, after lookup, call `MusicBrainzMapper.toTracklist(release.tracks)`.
