MusicBrainz currently stores tags as List&lt;String&gt; on models. Parser extractTags returns List&lt;String&gt; after sorting by count but discards the count. Need to preserve vote count for confidence calculation.

Current MusicBrainz tag extraction (MusicBrainzParser.kt):
```kotlin
internal fun extractTags(obj: JSONObject): List<String> {
    val tags = obj.optJSONArray("tags") ?: return emptyList()
    return (0 until tags.length())
        .map { i -> tags.getJSONObject(i) }
        .sortedByDescending { it.optInt("count", 0) }
        .map { it.getString("name") }
}
```

Last.fm currently returns tags as List&lt;String&gt; from parseTags. Tag position in the list indicates relevance.

Discogs model already has genres: List&lt;String&gt;? and styles: List&lt;String&gt;? on DiscogsRelease.
iTunes model has primaryGenreName: String? on ITunesAlbumResult.
