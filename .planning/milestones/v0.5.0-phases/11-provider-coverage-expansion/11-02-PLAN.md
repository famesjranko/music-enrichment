<objective>
Add iTunes lookup endpoints for ALBUM_TRACKS and ARTIST_DISCOGRAPHY at priority 30, with collectionId/artistId storage for precise lookups.

Purpose: iTunes currently only provides ALBUM_ART and ALBUM_METADATA from search results. The lookup API (`/lookup?id={id}&entity=song` and `/lookup?id={id}&entity=album`) returns structured track lists and artist discographies. Storing the collectionId and artistId from initial search enables direct lookups without re-searching. This adds fallback coverage behind MusicBrainz (100) and Deezer (50).

Output: Two new capabilities on ITunesProvider, two new API methods, two new model classes, two new mapper functions, ID storage via resolvedIdentifiers. All with tests.
</objective>
