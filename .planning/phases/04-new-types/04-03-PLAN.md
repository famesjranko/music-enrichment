**DeezerApi.kt** - Add 3 new methods:

1. `searchArtist(name: String): DeezerArtistSearchResult?` - Calls `GET /search/artist?q={name}&limit=1`. Parses `data[0]` for `id` and `name`. Returns null if no results. This is needed to get a Deezer artist ID for subsequent calls.

2. `getArtistAlbums(artistId: Long, limit: Int = 50): List<DeezerArtistAlbum>` - Calls `GET /artist/{id}/albums?limit={limit}`. Parses `data[]` array. Each entry has `id`, `title`, `release_date`, `record_type`, `cover_small`, `cover_medium`.

3. `getAlbumTracks(albumId: Long): List<DeezerTrack>` - Calls `GET /album/{id}/tracks`. Parses `data[]` array. Each entry has `id`, `title`, `track_position`, `duration` (in seconds).

For the Deezer search-to-ID pipeline: the provider will first search for the artist/album to get a Deezer ID, then use the ID-based endpoints. Store the Deezer ID in `identifiers.extra["deezerId"]` for future lookups.

**DeezerMapper.kt** - Add new mapper methods:

1. `toDiscography(albums: List<DeezerArtistAlbum>): EnrichmentData.Discography` - Map each to `DiscographyAlbum(title, year = releaseDate?.take(4), type = recordType, thumbnailUrl = coverMedium ?: coverSmall, identifiers = EnrichmentIdentifiers().withExtra("deezerId", id.toString()))`.

2. `toTracklist(tracks: List<DeezerTrack>): EnrichmentData.Tracklist` - Map each to `TrackInfo(title, position = trackPosition, durationMs = durationSec.toLong() * 1000, identifiers = EnrichmentIdentifiers().withExtra("deezerId", id.toString()))`.
