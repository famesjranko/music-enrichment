Add three new methods to the HttpClient interface, following the existing fetchJsonResult pattern:

1. `suspend fun fetchJsonArrayResult(url: String): HttpResult<JSONArray>` -- GET returning typed JSONArray result
2. `suspend fun postJsonResult(url: String, body: String): HttpResult<JSONObject>` -- POST returning typed JSONObject result
3. `suspend fun postJsonArrayResult(url: String, body: String): HttpResult<JSONArray>` -- POST returning typed JSONArray result

In DefaultHttpClient, implement each new method. Follow the exact pattern of the existing fetchJsonResult implementation:
- Wrap in `withContext(Dispatchers.IO)`
- Connect, read responseCode
- 429 -> HttpResult.RateLimited (parse Retry-After header * 1000)
- 400-499 -> HttpResult.ClientError(code, readErrorBody(conn))
- 500-599 -> HttpResult.ServerError(code, readErrorBody(conn))
- 200-299 -> read body, parse as JSONArray or JSONObject accordingly, wrap in HttpResult.Ok
- JSONException during parsing -> HttpResult.NetworkError("JSON parse error: ...")
- IOException -> HttpResult.NetworkError(e.message, e)

For POST methods: set requestMethod = "POST", Content-Type = "application/json", doOutput = true, write body bytes before reading response.

For fetchJsonArrayResult: same as fetchJsonResult but parse with `JSONArray(text)` instead of `JSONObject(text)`.

3. Override `fetchJsonArrayResult` -- check httpResultArrayResponses first, then fall back to fetchJsonArray auto-conversion (if non-null -> Ok, else -> NetworkError)
4. Override `postJsonResult` -- check httpResultResponses first, then fall back to postJson auto-conversion
5. Override `postJsonArrayResult` -- check httpResultArrayResponses first, then fall back to postJsonArray auto-conversion

private fun mapError(type: EnrichmentType, e: Exception): EnrichmentResult.Error {
    val kind = when (e) {
        is java.io.IOException -> ErrorKind.NETWORK
